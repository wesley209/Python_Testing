26 # Notes on reloading
27 # If your HttpFilterProc or HttpExtensionProc functions raises
28 # 'isapi.InternalReloadException', the framework will not treat it
29 # as an error but instead will terminate your extension, reload your
30 # extension module, re-initialize the instance, and re-issue the request.
31 # The Initialize functions are called with None as their param. The
32 # return code from the terminate function is ignored.
33 #
34 # This is all the framework does to help you. It is up to your code
35 # when you raise this exception. This sample uses a Win32 "find
36 # notification". Whenever windows tells us one of the files in the
37 # directory has changed, we check if the time of our source-file has
38 # changed, and set a flag. Next imcoming request, we check the flag and
39 # raise the special exception if set.
40 #
41 # The end result is that the module is automatically reloaded whenever
42 # the source-file changes - you need take no further action to see your
43 # changes reflected in the running server.
44  
45 # The framework only reloads your module - if you have libraries you
46 # depend on and also want reloaded, you must arrange for this yourself.
47 # One way of doing this would be to special case the import of these
48 # modules. Eg:
49 # --
50 # try:
51 # my_module = reload(my_module) # module already imported - reload it
52 # except NameError:
53 # import my_module # first time around - import it.
54 # --
55 # When your module is imported for the first time, the NameError will
56 # be raised, and the module imported. When the ISAPI framework reloads
57 # your module, the existing module will avoid the NameError, and allow
58 # you to reload that module.
