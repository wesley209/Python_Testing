83 # Notes on the approach:
84 #
85 # It's easy to wrap a selector implementation around
86 # ``gevent.select.poll``; in fact that's what happens by default
87 # when monkey-patching in Python 3. But the problem with that is
88 # each call to ``selector.select()`` will result in creating and
89 # then destroying new kernel-level polling resources, as nothing
90 # in ``gevent.select`` can keep watchers around (because the underlying
91 # file could be closed at any time). This ends up producing a large
92 # number of syscalls that are unnecessary.
93 #
94 # So here, we take advantage of the fact that it is documented and
95 # required that files not be closed while they are registered.
96 # This lets us persist watchers. Indeed, it lets us continually
97 # accrue events in the background before a call to ``select()`` is even
98 # made. We can take advantage of this to return results immediately, without
99 # a syscall, if we have them.
100 #
101 # We create watchers in ``register()`` and destroy them in
102 # ``unregister()``. They do not get started until the first call
103 # to ``select()``, though. Once they are started, they don't get
104 # stopped until they deliver an event.
105 # Lifecycle:
106 # register() -> inactive_watchers
107 # select() -> inactive_watchers -> active_watchers;
108 # active_watchers -> inactive_watchers
109
