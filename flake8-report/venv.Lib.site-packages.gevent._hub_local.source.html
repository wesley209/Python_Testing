28 # Use a class with an initializer so that we can test
29 # for 'is None' instead of catching AttributeError, making
30 # the code cleaner and possibly solving some corner cases
31 # (like #687).
32 #
33 # However, under some weird circumstances, it _seems_ like the
34 # __init__ method doesn't get called properly ("seems" is the
35 # keyword). We've seen at least one instance
36 # (https://github.com/gevent/gevent/issues/1961) of
37 # ``AttributeError: '_Threadlocal' object has no attribute # 'hub'``
38 # which should be impossible unless:
39 #
40 # - Someone manually deletes the attribute
41 # - The _threadlocal object itself is in the process of being
42 # deleted. The C ``tp_clear`` slot for it deletes the ``__dict__``
43 # of each instance in each thread (and/or the ``tp_clear`` of ``dict`` itself
44 # clears the instance). Now, how we could be getting
45 # cleared while still being used is unclear, but clearing is part of
46 # circular garbage collection, and in the bug report it looks like we're inside a
47 # weakref finalizer or ``__del__`` method, which could suggest that
48 # garbage collection is happening.
49 #
50 # See https://github.com/gevent/gevent/issues/1961
51 # and ``get_hub_if_exists()``
