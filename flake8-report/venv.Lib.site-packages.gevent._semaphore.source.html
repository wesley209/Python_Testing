48 class Semaphore(AbstractLinkable): # pylint:disable=undefined-variable
49 """
50 Semaphore(value=1) -> Semaphore
51  
52 .. seealso:: :class:`BoundedSemaphore` for a safer version that prevents
53 some classes of bugs. If unsure, most users should opt for `BoundedSemaphore`.
54  
55 A semaphore manages a counter representing the number of `release`
56 calls minus the number of `acquire` calls, plus an initial value.
57 The `acquire` method blocks if necessary until it can return
58 without making the counter negative. A semaphore does not track ownership
59 by greenlets; any greenlet can call `release`, whether or not it has previously
60 called `acquire`.
61  
62 If not given, ``value`` defaults to 1.
63  
64 The semaphore is a context manager and can be used in ``with`` statements.
65  
66 This Semaphore's ``__exit__`` method does not call the trace function
67 on CPython, but does under PyPy.
68  
69 .. versionchanged:: 1.4.0
70 Document that the order in which waiters are awakened is not specified. It was not
71 specified previously, but due to CPython implementation quirks usually went in FIFO order.
72 .. versionchanged:: 1.5a3
73 Waiting greenlets are now awakened in the order in which they waited.
74 .. versionchanged:: 1.5a3
75 The low-level ``rawlink`` method (most users won't use this) now automatically
76 unlinks waiters before calling them.
77 .. versionchanged:: 20.12.0
78 Improved support for multi-threaded usage. When multi-threaded usage is detected,
79 instances will no longer create the thread's hub if it's not present.
80  
81 .. versionchanged:: 24.2.1
82 Uses Python 3 native lock timeouts for cross-thread operations instead
83 of spinning.
