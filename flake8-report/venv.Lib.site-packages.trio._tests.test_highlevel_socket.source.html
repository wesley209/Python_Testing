72 # Check a send_all that has to be split into multiple parts (on most
73 # platforms... on Windows every send() either succeeds or fails as a
74 # whole)
75 async def sender() -> None:
76 data = bytearray(BIG)
77 await a.send_all(data)
78 # send_all uses memoryviews internally, which temporarily "lock"
79 # the object they view. If it doesn't clean them up properly, then
80 # some bytearray operations might raise an error afterwards, which
81 # would be a pretty weird and annoying side-effect to spring on
82 # users. So test that this doesn't happen, by forcing the
83 # bytearray's underlying buffer to be realloc'ed:
84 data += bytes(BIG)
85 # (Note: the above line of code doesn't do a very good job at
86 # testing anything, because:
87 # - on CPython, the refcount GC generally cleans up memoryviews
88 # for us even if we're sloppy.
89 # - on PyPy3, at least as of 5.7.0, the memoryview code and the
90 # bytearray code conspire so that resizing never fails – if
91 # resizing forces the bytearray's internal buffer to move, then
92 # all memoryview references are automagically updated (!!).
93 # See:
94 # https://gist.github.com/njsmith/0ffd38ec05ad8e34004f34a7dc492227
95 # But I'm leaving the test here in hopes that if this ever changes
96 # and we break our implementation of send_all, then we'll get some
97 # early warning...)
