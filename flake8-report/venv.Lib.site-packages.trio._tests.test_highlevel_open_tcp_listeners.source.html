378 # We used to have an elaborate test that opened a real TCP listening socket
379 # and then tried to measure its backlog by making connections to it. And most
380 # of the time, it worked. But no matter what we tried, it was always fragile,
381 # because it had to do things like use timeouts to guess when the listening
382 # queue was full, sometimes the CI hosts go into SYN-cookie mode (where there
383 # effectively is no backlog), sometimes the host might not be enough resources
384 # to give us the full requested backlog... it was a mess. So now we just check
385 # that the backlog argument is passed through correctly.

394 ]:
