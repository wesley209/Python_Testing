27 class Waiter(object):
28 """
29 A low level communication utility for greenlets.
30  
31 Waiter is a wrapper around greenlet's ``switch()`` and ``throw()`` calls that makes them somewhat safer:
32  
33 * switching will occur only if the waiting greenlet is executing :meth:`get` method currently;
34 * any error raised in the greenlet is handled inside :meth:`switch` and :meth:`throw`
35 * if :meth:`switch`/:meth:`throw` is called before the receiver calls :meth:`get`, then :class:`Waiter`
36 will store the value/exception. The following :meth:`get` will return the value/raise the exception.
37  
38 The :meth:`switch` and :meth:`throw` methods must only be called from the :class:`Hub` greenlet.
39 The :meth:`get` method must be called from a greenlet other than :class:`Hub`.
40  
41 >>> from gevent.hub import Waiter
42 >>> from gevent import get_hub
43 >>> result = Waiter()
44 >>> timer = get_hub().loop.timer(0.1)
45 >>> timer.start(result.switch, 'hello from Waiter')
46 >>> result.get() # blocks for 0.1 seconds
47 'hello from Waiter'
48 >>> timer.close()
49  
50 If switch is called before the greenlet gets a chance to call :meth:`get` then
51 :class:`Waiter` stores the value.
52  
53 >>> from gevent.time import sleep
54 >>> result = Waiter()
55 >>> timer = get_hub().loop.timer(0.1)
56 >>> timer.start(result.switch, 'hi from Waiter')
57 >>> sleep(0.2)
58 >>> result.get() # returns immediately without blocking
59 'hi from Waiter'
60 >>> timer.close()
61  
62 .. warning::
63  
64 This is a limited and dangerous way to communicate between
65 greenlets. It can easily leave a greenlet unscheduled forever
66 if used incorrectly. Consider using safer classes such as
67 :class:`gevent.event.Event`, :class:`gevent.event.AsyncResult`,
68 or :class:`gevent.queue.Queue`.
69 """
