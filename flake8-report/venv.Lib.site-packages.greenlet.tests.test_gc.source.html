49 # This test is designed to crash when active greenlets
50 # are made garbage collectable, until the underlying
51 # problem is resolved. How does it work:
52 # - order of object creation is important
53 # - array is created first, so it is moved to unreachable first
54 # - we create a cycle between a greenlet and this array
55 # - we create an object that participates in gc, is only
56 # referenced by a greenlet, and would corrupt gc lists
57 # on destruction, the easiest is to use an object with
58 # a finalizer
59 # - because array is the first object in unreachable it is
60 # cleared first, which causes all references to greenlet
61 # to disappear and causes greenlet to be destroyed, but since
62 # it is still live it causes a switch during gc, which causes
63 # an object with finalizer to be destroyed, which causes stack
64 # corruption and then a crash
