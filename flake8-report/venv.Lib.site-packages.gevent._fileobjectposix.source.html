258 class FileObjectPosix(FileObjectBase):
259 """
260 FileObjectPosix()
261  
262 A file-like object that operates on non-blocking files but
263 provides a synchronous, cooperative interface.
264  
265 .. caution::
266 This object is only effective wrapping files that can be used meaningfully
267 with :func:`select.select` such as sockets and pipes.
268  
269 In general, on most platforms, operations on regular files
270 (e.g., ``open('a_file.txt')``) are considered non-blocking
271 already, even though they can take some time to complete as
272 data is copied to the kernel and flushed to disk: this time
273 is relatively bounded compared to sockets or pipes, though.
274 A :func:`~os.read` or :func:`~os.write` call on such a file
275 will still effectively block for some small period of time.
276 Therefore, wrapping this class around a regular file is
277 unlikely to make IO gevent-friendly: reading or writing large
278 amounts of data could still block the event loop.
279  
280 If you'll be working with regular files and doing IO in large
281 chunks, you may consider using
282 :class:`~gevent.fileobject.FileObjectThread` or
283 :func:`~gevent.os.tp_read` and :func:`~gevent.os.tp_write` to bypass this
284 concern.
285  
286 .. tip::
287 Although this object provides a :meth:`fileno` method and so
288 can itself be passed to :func:`fcntl.fcntl`, setting the
289 :data:`os.O_NONBLOCK` flag will have no effect (reads will
290 still block the greenlet, although other greenlets can run).
291 However, removing that flag *will cause this object to no
292 longer be cooperative* (other greenlets will no longer run).
293  
294 You can use the internal ``fileio`` attribute of this object
295 (a :class:`io.RawIOBase`) to perform non-blocking byte reads.
296 Note, however, that once you begin directly using this
297 attribute, the results from using methods of *this* object
298 are undefined, especially in text mode. (See :issue:`222`.)
299  
300 .. versionchanged:: 1.1
301 Now uses the :mod:`io` package internally. Under Python 2, previously
302 used the undocumented class :class:`socket._fileobject`. This provides
303 better file-like semantics (and portability to Python 3).
304 .. versionchanged:: 1.2a1
305 Document the ``fileio`` attribute for non-blocking reads.
306 .. versionchanged:: 1.2a1
307  
308 A bufsize of 0 in write mode is no longer forced to be 1.
309 Instead, the underlying buffer is flushed after every write
310 operation to simulate a bufsize of 0. In gevent 1.0, a
311 bufsize of 0 was flushed when a newline was written, while
312 in gevent 1.1 it was flushed when more than one byte was
313 written. Note that this may have performance impacts.
314 .. versionchanged:: 1.3a1
315 On Python 2, enabling universal newlines no longer forces unicode
316 IO.
317 .. versionchanged:: 1.5
318 The default value for *mode* was changed from ``rb`` to ``r``. This is consistent
319 with :func:`open`, :func:`io.open`, and :class:`~.FileObjectThread`, which is the
320 default ``FileObject`` on some platforms.
321 .. versionchanged:: 1.5
322 Stop forcing buffering. Previously, given a ``buffering=0`` argument,
323 *buffering* would be set to 1, and ``buffering=1`` would be forced to
324 the default buffer size. This was a workaround for a long-standing concurrency
325 issue. Now the *buffering* argument is interpreted as intended.
326 """
