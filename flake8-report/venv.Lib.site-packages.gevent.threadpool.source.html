244 class ThreadPool(GroupMappingMixin):
245 """
246 A pool of native worker threads.
247  
248 This can be useful for CPU intensive functions, or those that
249 otherwise will not cooperate with gevent. The best functions to execute
250 in a thread pool are small functions with a single purpose; ideally they release
251 the CPython GIL. Such functions are extension functions implemented in C.
252  
253 It implements the same operations as a :class:`gevent.pool.Pool`,
254 but using threads instead of greenlets.
255  
256 .. note:: The method :meth:`apply_async` will always return a new
257 greenlet, bypassing the threadpool entirely.
258  
259 Most users will not need to create instances of this class. Instead,
260 use the threadpool already associated with gevent's hub::
261  
262 pool = gevent.get_hub().threadpool
263 result = pool.spawn(lambda: "Some func").get()
264  
265 .. important:: It is only possible to use instances of this class from
266 the thread running their hub. Typically that means from the thread that
267 created them. Using the pattern shown above takes care of this.
268  
269 There is no gevent-provided way to have a single process-wide limit on the
270 number of threads in various pools when doing that, however. The suggested
271 way to use gevent and threadpools is to have a single gevent hub
272 and its one threadpool (which is the default without doing any extra work).
273 Only dispatch minimal blocking functions to the threadpool, functions that
274 do not use the gevent hub.
275  
276 The `len` of instances of this class is the number of enqueued
277 (unfinished) tasks.
278  
279 Just before a task starts running in a worker thread,
280 the values of :func:`threading.setprofile` and :func:`threading.settrace`
281 are consulted. Any values there are installed in that thread for the duration
282 of the task (using :func:`sys.setprofile` and :func:`sys.settrace`, respectively).
283 (Because worker threads are long-lived and outlast any given task, this arrangement
284 lets the hook functions change between tasks, but does not let them see the
285 bookkeeping done by the worker thread itself.)
286  
287 .. caution:: Instances of this class are only true if they have
288 unfinished tasks.
289  
290 .. versionchanged:: 1.5a3
291 The undocumented ``apply_e`` function, deprecated since 1.1,
292 was removed.
293 .. versionchanged:: 20.12.0
294 Install the profile and trace functions in the worker thread while
295 the worker thread is running the supplied task.
296 .. versionchanged:: 22.08.0
297 Add the option to let idle threads expire and be removed
298 from the pool after *idle_task_timeout* seconds (-1 for no
299 timeout)
300 """
