2 #
3 # generic "output window"
4 #
5 # This Window will detect itself closing, and recreate next time output is
6 # written to it.
7  
8 # This has the option of writing output at idle time (by hooking the
9 # idle message, and queueing output) or writing as each
10 # write is executed.
11 # Updating the window directly gives a jerky appearance as many writes
12 # take place between commands, and the windows scrolls, and updates etc
13 # Updating at idle-time may defer all output of a long process, giving the
14 # appearence nothing is happening.
15 # There is a compromise "line" mode, which will output whenever
16 # a complete line is available.
17  
18 # behaviour depends on self.writeQueueing
19  
20 # This module is thread safe - output can originate from any thread. If any thread
21 # other than the main thread attempts to print, it is always queued until next idle time
22
