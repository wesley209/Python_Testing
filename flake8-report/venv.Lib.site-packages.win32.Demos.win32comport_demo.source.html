1 # This is a simple serial port terminal demo.
2 #
3 # Its primary purpose is to demonstrate the native serial port access offered via
4 # win32file.
5  
6 # It uses 3 threads:
7 # - The main thread, which cranks up the other 2 threads, then simply waits for them to exit.
8 # - The user-input thread - blocks waiting for a keyboard character, and when found sends it
9 # out the COM port. If the character is Ctrl+C, it stops, signalling the COM port thread to stop.
10 # - The COM port thread is simply listening for input on the COM port, and prints it to the screen.
11  
12 # This demo uses userlapped IO, so that none of the read or write operations actually block (however,
13 # in this sample, the very next thing we do _is_ block - so it shows off the concepts even though it
14 # doesn't exploit them.
