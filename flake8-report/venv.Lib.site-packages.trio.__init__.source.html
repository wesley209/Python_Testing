8 # General layout:
9 #
10 # trio/_core/... is the self-contained core library. It does various
11 # shenanigans to export a consistent "core API", but parts of the core API are
12 # too low-level to be recommended for regular use.
13 #
14 # trio/*.py define a set of more usable tools on top of this. They import from
15 # trio._core and from each other.
16 #
17 # This file pulls together the friendly public API, by re-exporting the more
18 # innocuous bits of the _core API + the higher-level tools from trio/*.py.
19 #
20 # Uses `from x import y as y` for compatibility with `pyright --verifytypes` (#2625)
