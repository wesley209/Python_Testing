21 """
22 This function is the decorator which is used to wrap a Flask route with.
23 In the simplest case, simply use the default parameters to allow all
24 origins in what is the most permissive configuration. If this method
25 modifies state or performs authentication which may be brute-forced, you
26 should add some degree of protection, such as Cross Site Request Forgery
27 protection.
28  
29 :param origins:
30 The origin, or list of origins to allow requests from.
31 The origin(s) may be regular expressions, case-sensitive strings,
32 or else an asterisk
33  
34 Default : '*'
35 :type origins: list, string or regex
36  
37 :param methods:
38 The method or list of methods which the allowed origins are allowed to
39 access for non-simple requests.
40  
41 Default : [GET, HEAD, POST, OPTIONS, PUT, PATCH, DELETE]
42 :type methods: list or string
43  
44 :param expose_headers:
45 The header or list which are safe to expose to the API of a CORS API
46 specification.
47  
48 Default : None
49 :type expose_headers: list or string
50  
51 :param allow_headers:
52 The header or list of header field names which can be used when this
53 resource is accessed by allowed origins. The header(s) may be regular
54 expressions, case-sensitive strings, or else an asterisk.
55  
56 Default : '*', allow all headers
57 :type allow_headers: list, string or regex
58  
59 :param supports_credentials:
60 Allows users to make authenticated requests. If true, injects the
61 `Access-Control-Allow-Credentials` header in responses. This allows
62 cookies and credentials to be submitted across domains.
63  
64 :note: This option cannot be used in conjunction with a '*' origin
65  
66 Default : False
67 :type supports_credentials: bool
68  
69 :param max_age:
70 The maximum time for which this CORS request maybe cached. This value
71 is set as the `Access-Control-Max-Age` header.
72  
73 Default : None
74 :type max_age: timedelta, integer, string or None
75  
76 :param send_wildcard: If True, and the origins parameter is `*`, a wildcard
77 `Access-Control-Allow-Origin` header is sent, rather than the
78 request's `Origin` header.
79  
80 Default : False
81 :type send_wildcard: bool
82  
83 :param vary_header:
84 If True, the header Vary: Origin will be returned as per the W3
85 implementation guidelines.
86  
87 Setting this header when the `Access-Control-Allow-Origin` is
88 dynamically generated (e.g. when there is more than one allowed
89 origin, and an Origin than '*' is returned) informs CDNs and other
90 caches that the CORS headers are dynamic, and cannot be cached.
91  
92 If False, the Vary header will never be injected or altered.
93  
94 Default : True
95 :type vary_header: bool
96  
97 :param automatic_options:
98 Only applies to the `cross_origin` decorator. If True, Flask-CORS will
99 override Flask's default OPTIONS handling to return CORS headers for
100 OPTIONS requests.
101  
102 Default : True
103 :type automatic_options: bool
104
