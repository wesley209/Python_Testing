19 # We need to protect imports both across threads and across greenlets.

21 # and imp module are deprecated. It seems that in all Py3 versions, a
22 # module lock is used such that this fix is not necessary.

27 # could lead to a LoopExit error as a greenlet attempts to block on it while
28 # it's already held by the main greenlet (issue #798).

40 # It will be removed when no longer needed.
41 # Nothing in this function yields, so we're multi-greenlet safe
42 # (But not multi-threading safe.)
43 # XXX: What about on PyPy, where the GC is asynchronous (not ref-counting)?
44 # (Does it stop-the-world first?)
45 lock = None

47 lock = _g_import_locks[name]()
